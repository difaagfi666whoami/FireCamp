## Keputusan Go/No-Go

### Aturan Scoring

| Kondisi | Keputusan |
|---|---|
| **6+ sinyal 🟢** | ✅ **PROCEED** — Onboard ke early access. Ini adalah pengguna ideal. |
| **4-5 sinyal 🟢 (atau 3 🟢 + banyak 🟡)** | ⚠️ **PROCEED DENGAN CATATAN** — Ada pain nyata tapi ada gap. Catat apa yang kurang dan address sebelum jalan lebih jauh. |
| **2-3 sinyal 🟢** | 🟡 **TAHAN** — Pain belum cukup kuat atau ini bukan target user yang tepat. Revisit positioning sebelum follow-up. |
| **0-1 sinyal 🟢** | ❌ **NO-GO** — Jangan deploy waktu untuk ini. Pelajari kenapa tidak resonan dan iterasi hipotesis. |
